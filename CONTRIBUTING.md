# Contributing to Create: Angels & Afterburners

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to **Create: Angels & Afterburners**.

## How Can I Contribute?

### Reporting Bugs
* Use the GitHub Issue Tracker.
* Provide a clear and descriptive title.
* Describe the steps to reproduce the bug.
* Attach logs or screenshots if possible.

### Suggesting Enhancements
* Open an issue to discuss your idea.
* Be as descriptive as possible about the feature and why it would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the project builds locally using `./gradlew build`.
4. Follow the existing code style.
5. Provide a clear description of the changes in your PR.

## Code Style
* Follow standard Java naming conventions.
* Use 4 spaces for indentation.
* Add Javadoc for complex logic or public APIs.

Refer to the [README](README.md) for setup instructions.
